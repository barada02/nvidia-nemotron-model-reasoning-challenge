# Strategies for Improving Nemotron Reasoning (Single-Pass Inference)

Because the competition evaluates the adapter using **greedy decoding** (`temperature = 0.0`, `top_p = 1.0`), test-time compute strategies like Self-Consistency (majority voting) are impossible. The model must produce the exact correct chain-of-thought and answer on the **very first try**.

To improve single-pass accuracy, we must focus on upgrading the **quality, diversity, and alignment** of the training data. Here are the three detailed paths you can work on individually.

---

## Path 1: Iterative SFT (Rejection Sampling / Self-Play)

**The Concept:** The current training data relies 100% on what the author's Python scripts could solve. There are 918 problems marked `rule_unknown`. However, your newly trained Unsloth adapter has generalized reasoning skills. It might be able to solve some of those 918 problems purely through inference! 

If the model gets it right, its reasoning trace is valid training data.

### Step-by-Step Implementation:
1. **Filter Unsolved Problems:** Read `problems.jsonl` and extract the IDs of all problems with `status: rule_unknown` (or cross-reference `train.csv` against the `corpus/` directory).
2. **Run Inference:** Write an inference script using your Unsloth adapter to generate answers for these unsolved problems. (Since you are generating data locally, you *can* use `temperature > 0` here to generate multiple attempts per problem).
3. **Verify:** Extract the `\boxed{answer}` from the model's output and compare it to the ground-truth answer in `train.csv`.
4. **Harvest:** If the model gets the answer correct, take that specific CoT trace, format it, and append it to your training corpus. 
5. **Retrain:** Run your `kaggle-finetuning.ipynb` notebook again on the expanded dataset. The model will learn from its own successful discoveries.

---

## Path 2: Expand the Deterministic Python Solvers

**The Concept:** The absolute best data for this model is the programmatic, flawless CoT traces generated by the Python solvers. The category `cryptarithm_deduce` alone has **559** unsolved problems. If you write a Python script that solves them, you instantly unlock a massive chunk of perfect training data.

### Step-by-Step Implementation:
1. **Pick a Target:** Open the `investigations/` directory and look at the `hypothesis_formed` files for `cryptarithm_deduce` or `equation_numeric_guess`. Understand the underlying logic of the puzzles the author couldn't automate.
2. **Write the Solver:** Create or modify a solver in `nemotron/reasoners/`. The solver must do two things:
   - Search for the correct logical rule that maps the examples' inputs to their outputs.
   - Print out a highly detailed, step-by-step text trace of *how* it deduced the rule (just like `bit_manipulation.py` does).
3. **Verify locally:** Run `reasoning.py` for your new solver. Ensure it produces the correct `\boxed{}` answer matching `train.csv`.
4. **Tokenize:** Run `corpus.py` to rebuild the corpus with your thousands of new, flawless reasoning tokens.

---

## Path 3: Direct Preference Optimization (DPO)

**The Concept:** Supervised Fine-Tuning (SFT) teaches the model *how* to solve a problem, but it doesn't teach it how to recover from mistakes or what logical dead-ends look like. DPO trains the model using preferences: "Do this, don't do that."

### Step-by-Step Implementation:
1. **Generate Negative Examples:** Write a script to run inference on the **already solved** training set problems. Use `temperature = 0.7` to force the model to occasionally make logical errors or arithmetic mistakes that lead to an incorrect `\boxed{}` answer.
2. **Create Preference Pairs:** For a given problem, you now have:
   - **Chosen (Winner):** The perfect Python-generated CoT from the author's `corpus/`.
   - **Rejected (Loser):** The flawed CoT generated by your model that got the wrong answer.
3. **Format for Unsloth:** Unsloth supports `DPOTrainer` via HuggingFace `trl`. Format your dataset into `{prompt, chosen, rejected}` dictionaries.
4. **Train:** Run DPO training on top of your SFT adapter. This will aggressively penalize the model for making the specific types of arithmetic or logical errors present in the "Rejected" traces, vastly improving its robustness at `temperature = 0.0`.
